# UI 模版化开发方案

## 核心理念

### 传统开发 vs 模版化开发

| 传统开发 | 模版化开发 |
|---------|-----------|
| 写页面 → 做功能 | 选模版 → 组模版 → 配功能 |
| 每个页面单独实现 | 模版可复用、可组合 |
| 修改需要重写代码 | 修改只需换模版配置 |
| 一套界面 | 无限组合可能 |

### 设计原则

1. **模版优先**：所有 UI 都是模版，不是固定页面
2. **可组合性**：模版可以嵌套组合
3. **可替换性**：同类型模版可以互换
4. **配置驱动**：通过配置数据驱动模版展示

## 模版分类体系

### 一级：页面框架模版 (Page Framework)

定义整个页面的基础结构

#### 已实现的示例
- `NestedRoutesPage` - 多级嵌套路由页面 (在 example 中)

#### 计划中的框架模版
```
1. 单列框架
   - 顶部导航 + 内容区域
   - 用途: 详情页、设置页

2. 双列框架
   - 左侧分类 + 右侧内容
   - 用途: 商品分类、内容分类

3. 三级嵌套框架
   - 顶部分类 → 中间标签 → 底部内容
   - 用途: 复杂内容展示

4. 四级嵌套框架
   - 顶部滑动 → 一级标签 → 二级标签 → 内容区
   - 用途: 超复杂分类展示 (example 已实现)

5. TabBar 框架
   - 底部/顶部 Tab + 多个页面
   - 用途: 主页导航

6. 抽屉框架
   - 侧边栏抽屉 + 主内容区
   - 用途: 设置、导航

7. 分层框架
   - 悬浮层 + 底层内容
   - 用途: 视频播放、地图
```

### 二级：布局模版 (Layout Template)

定义内容区域的排列方式

```
1. 瀑布流布局
   - 错落卡片显示
   - 支持 2/3/4 列
   - 用途: 图片、商品、内容卡片

2. 网格布局
   - 等高等宽网格
   - 支持 2/3/4 列
   - 用途: 图标、功能入口

3. 列表布局
   - 垂直列表
   - 支持分组、分割线
   - 用途: 消息、设置、内容列表

4. 轮播布局
   - 水平滑动
   - 支持自动播放、指示器
   - 用途: Banner、广告

5. 标签页布局
   - 横向滑动标签
   - 支持动态添加
   - 用途: 分类、筛选

6. 时间轴布局
   - 垂直时间线
   - 支持节点样式
   - 用途: 动态、订单记录

7. 卡片堆叠布局
   - 卡片叠加
   - 支持滑动翻页
   - 用途: 信息流、推荐
```

### 三级：组件模版 (Component Template)

定义单个 UI 单元的样式

```
1. 内容卡片组件
   - 标题 + 图片 + 文本 + 元数据
   - 用途: 文章、帖子、商品

2. 用户卡片组件
   - 头像 + 昵称 + 简介 + 关注按钮
   - 用途: 用户列表、推荐

3. 消息卡片组件
   - 发送者 + 内容 + 时间
   - 用途: 聊天消息

4. 操作卡片组件
   - 图标 + 标题 + 描述
   - 用途: 功能入口

5. 图片卡片组件
   - 大图 + 标签
   - 用途: 图片展示

6. 视频卡片组件
   - 缩略图 + 时长 + 标题
   - 用途: 视频列表

7. 商品卡片组件
   - 图片 + 价格 + 标题 + 购买按钮
   - 用途: 商品列表
```

### 四级：元素模版 (Element Template)

定义最小 UI 单元

```
1. 按钮元素 (主要、次要、文字、图标按钮)
2. 输入框元素 (文本、密码、搜索)
3. 标签元素 (分类标签、状态标签)
4. 徽章元素 (数字徽章、红点)
5. 头像元素 (圆形、方形、组头像)
6. 进度元素 (进度条、环形进度)
7. 分割元素 (分割线、间隙)
```

## 模版组合方式

### 示例 1: 首页组合 A

```
四级嵌套框架
├─ 顶部: 轮播布局 (Banner)
├─ 一级标签: [推荐, 关注, 热门]
│  ├─ 推荐
│  │  └─ 二级标签: [全部, 视频, 图文]
│  │     └─ 全部
│  │        └─ 瀑布流布局
│  │           ├─ 内容卡片组件
│  │           ├─ 视频卡片组件
│  │           └─ 商品卡片组件
│  ├─ 关注
│  │  └─ 列表布局
│  │     └─ 用户卡片组件
│  └─ 热门
│     └─ 网格布局
│        └─ 话题卡片组件
└─ 底部: TabBar 导航
```

```
双列框架
├─ 左侧: 分类列表
└─ 右侧: 瀑布流布局
   └─ 商品卡片组件
```

### 示例 3: 详情页组合

```
单列框架
├─ 顶部: 视频播放器组件
├─ 中部: 列表布局
│  ├─ 标题卡片组件
```

## 配置驱动设计

### 模版配置示例

```dart
// 页面配置
class PageConfig {
  final String id;
  final PageFrameworkType framework;
  final List<SectionConfig> sections;
}

// 区块配置
class SectionConfig {
  final String id;
  final LayoutType layout;
  final ComponentType component;
  final Map<String, dynamic> params;
  final DataSourceConfig dataSource;
}

// 数据源配置
class DataSourceConfig {
  final String service;      // 如: service_content
  final String method;       // 如: getContents
  final Map<String, dynamic> params;
}
```

### 配置示例: 首页

```dart
final homePageConfig = PageConfig(
  id: 'home_page_v1',
  framework: PageFrameworkType.fourLevelNested,
  sections: [
    SectionConfig(
      id: 'banner_section',
      layout: LayoutType.carousel,
      component: ComponentType.imageCard,
      params: {
        'autoPlay': true,
        'interval': 3000,
      },
      dataSource: DataSourceConfig(
        service: 'service_content',
        method: 'getBanners',
      ),
    ),
    SectionConfig(
      id: 'recommend_section',
      layout: LayoutType.waterfall,
      component: ComponentType.contentCard,
      params: {
        'columns': 2,
      },
      dataSource: DataSourceConfig(
        service: 'service_content',
        method: 'getContents',
        params: {
          'sortType': 'recommend',
        },
      ),
    ),
  ],
);
```

## 实现方案

### Package 结构

```
packages/
├── ui/
│   ├── ui_templates/           # 模版库
│   │   ├── lib/
│   │   │   ├── frameworks/     # 页面框架
│   │   │   ├── layouts/        # 布局模版
│   │   │   ├── components/     # 组件模版
│   │   │   └── elements/       # 元素模版
│   │   └── pubspec.yaml
│   │
│   ├── ui_builder/             # 模版构建器
│   │   ├── lib/
│   │   │   ├── builders/       # 构建器
│   │   │   ├── config/         # 配置模型
│   │   │   └── widgets/        # 通用 Widget
│   │   └── pubspec.yaml
│   │
│   └── ui_examples/            # 模版示例
│       ├── lib/
│       │   ├── pages/          # 示例页面
│       │   └── configs/        # 示例配置
│       └── pubspec.yaml
```

### 关键技术

1. **配置驱动**: 使用配置文件定义页面结构
2. **动态加载**: 根据配置动态加载模版
3. **数据绑定**: 自动对接 Service 层数据
4. **热更新**: 配置可远程下发 (可选)

## 开发流程

### 1. 创建模版

```dart
// packages/ui_templates/lib/layouts/waterfall_layout.dart
class WaterfallLayout extends StatelessWidget {
  final List<Widget> children;
  final int columns;
  final double spacing;

  const WaterfallLayout({
    required this.children,
    this.columns = 2,
    this.spacing = 8,
  });

  @override
  Widget build(BuildContext context) {
    return MasonryGridView.count(
      crossAxisCount: columns,
      mainAxisSpacing: spacing,
      crossAxisSpacing: spacing,
      children: children,
    );
  }
}
```

### 2. 注册模版

```dart
// packages/ui_builder/lib/templates/template_registry.dart
class TemplateRegistry {
  static final Map<String, WidgetBuilder> layouts = {
    'waterfall': (params) => WaterfallLayout(...),
    'grid': (params) => GridLayout(...),
    'list': (params) => ListLayout(...),
  };
}
```

### 3. 使用配置

```dart
// packages/ui_examples/lib/pages/home_page.dart
class HomePage extends StatelessWidget {
  final PageConfig config;

  @override
  Widget build(BuildContext context) {
    return TemplateBuilder(config: config);
  }
}
```

## 优势

1. **快速开发**: 新页面只需配置，不需要写代码
2. **易于修改**: 换个模版就是全新页面
3. **统一风格**: 复用模版保证 UI 一致
4. **A/B 测试**: 不同配置轻松对比
5. **远程配置**: 可实现无发版更新 UI
