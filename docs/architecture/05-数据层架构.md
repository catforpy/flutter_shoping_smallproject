## 1. 数据层概述

### 1.1 数据层架构图

```
┌───────────────────────────────────────────────┐
│                  UI Layer                    │
│          (Widgets, Pages, Screens)           │
└───────────────────────────────────────────────┘
                     ↓
┌───────────────────────────────────────────────┐
│                Provider Layer                │
│         (Riverpod Providers/Notifiers)       │
└───────────────────────────────────────────────┘
                     ↓
┌───────────────────────────────────────────────┐
│                 UseCase Layer                │
│         (Business Logic, Validation)         │
└───────────────────────────────────────────────┘
                     ↓
┌───────────────────────────────────────────────┐
│              Repository Layer                │
│         (Data Caching, Coordination)         │
└───────────────────────────────────────────────┘
                     ↓
        ┌────────────┴────────────┐
        ↓                          ↓
┌──────────────────┐      ┌──────────────────┐
```

## 2. 数据流向设计

### 2.1 读取流程

```
UI → Provider → UseCase → Repository
                              ↓
                         检查内存缓存
                              ↓ (未命中)
                         检查持久化缓存
                              ↓ (未命中)
                         检查数据库
                              ↓ (未命中)
                         请求远程API
                              ↓
                         保存到数据库
                              ↓
                         保存到缓存
                              ↓
                         返回数据
```

### 2.2 写入流程

```
UI → Provider → UseCase → Repository
                              ↓
                         请求远程API
                              ↓
                         更新数据库
                              ↓
                         更新缓存
                              ↓
                         返回结果
```

### 2.3 数据流示例代码

```dart
// Repository 实现
class UserRepositoryImpl implements UserRepository {
  final UserRemoteDataSource _remoteDataSource;
  final UserLocalDataSource _localDataSource;
  final CacheManager _cache;

  UserRepositoryImpl({
    required UserRemoteDataSource remoteDataSource,
    required UserLocalDataSource localDataSource,
    required CacheManager cache,
  })  : _remoteDataSource = remoteDataSource,
        _localDataSource = localDataSource,
        _cache = cache;

  @override
  Future<UserModel> getUser(String userId) async {
    // 1. 尝试从缓存获取
    final cacheKey = 'user_$userId';
    final cached = await _cache.get<UserModel>(cacheKey);
    if (cached != null) {
      return cached;
    }

    // 2. 从本地数据库获取
    final local = await _localDataSource.getUser(userId);
    if (local != null) {
      await _cache.set(cacheKey, local);
      return local;
    }

    // 3. 从远程获取
    final remote = await _remoteDataSource.fetchUser(userId);

    // 4. 保存到本地和缓存
    await Future.wait([
      _localDataSource.saveUser(remote),
      _cache.set(cacheKey, remote),
    ]);

    return remote;
  }
}
```

### 3.2 数据库版本管理

```dart
// core_database/lib/src/app_database.dart
/// 应用数据库
@DriftDatabase(tables: [
  Users,
  Products,
  Orders,
  Messages,
  Videos,
  Articles,
])
class AppDatabase extends _$AppDatabase {
  AppDatabase(QueryExecutor e) : super(e);

  @override
  int get schemaVersion => 1;

  @override
  MigrationStrategy get migration {
    return MigrationStrategy(
      onCreate: (Migrator m) async {
        // 创建所有表
        await m.createAll();
      },

      onUpgrade: (Migrator m, int from, int to) async {
        // 数据库升级
        if (from == 1 && to == 2) {
          // 添加新字段
          await m.addColumn(users, users.level);
        }
      },

      beforeOpen: (OpeningDetails details) async {
        // 启用外键约束
        await customStatement('PRAGMA foreign_keys = ON');
      },
    );
  }

  /// 清空所有数据
  Future<void> clearAll() async {
    // 按依赖顺序删除
    await delete(messages).go();
    await delete(orders).go();
    await delete(products).go();
    await delete(articles).go();
    await delete(videos).go();
    await delete(users).go();
  }
}
```

## 4. 缓存策略

### 4.1 三级缓存架构

```
┌─────────────────────────────────────────┐
│         L1: 内存缓存                     │
│  - 容量: 100MB                          │
│  - 速度: 极快 (~1ms)                     │
│  - 策略: LRU                            │
│  - 有效期: 应用生命周期                  │
└─────────────────────────────────────────┘
              ↓ miss
┌─────────────────────────────────────────┐
│         L2: 持久化缓存                   │
│  - 容量: 500MB                          │
│  - 速度: 快 (~10ms)                      │
│  - 策略: LFU                            │
│  - 有效期: 7天                          │
└─────────────────────────────────────────┘
              ↓ miss
┌─────────────────────────────────────────┐
│         L3: 数据库                       │
│  - 容量: 无限制                          │
│  - 速度: 中等 (~50ms)                    │
│  - 策略: 全量存储                       │
│  - 有效期: 永久                         │
└─────────────────────────────────────────┘
```

### 5.2 Repository 实现

```dart
// service_user/lib/src/repository/user_repository_impl.dart
/// 用户仓库实现
class UserRepositoryImpl implements UserRepository {
  final UserRemoteDataSource _remoteDataSource;
  final UserLocalDataSource _localDataSource;
  final CacheManager _cache;

  UserRepositoryImpl({
    required UserRemoteDataSource remoteDataSource,
    required UserLocalDataSource localDataSource,
    required CacheManager cache,
  })  : _remoteDataSource = remoteDataSource,
        _localDataSource = localDataSource,
        _cache = cache;

  @override
  Future<Result<UserModel>> getUser(String userId) async {
    try {
      // 1. 尝试从缓存获取
      final cacheKey = CacheKeys.user(userId);
      final cached = await _cache.get<UserModel>(cacheKey);
      if (cached != null) {
        return Result.success(cached);
      }

      // 2. 从本地数据库获取
      final local = await _localDataSource.getUser(userId);
      if (local != null) {
        await _cache.set(cacheKey, local, duration: CacheConfig.getCacheDuration('user'));
        return Result.success(local);
      }

      // 3. 从远程获取
      final remote = await _remoteDataSource.fetchUser(userId);

      // 4. 保存到本地和缓存
      await Future.wait([
        _localDataSource.saveUser(remote),
        _cache.set(cacheKey, remote, duration: CacheConfig.getCacheDuration('user')),
      ]);

      return Result.success(remote);
    } catch (e) {
      return Result.failure(e as Exception);
    }
  }

  @override
  Future<Result<Pagination<UserModel>>> getUserList({
    required int page,
    required int pageSize,
  }) async {
    try {
      final cacheKey = CacheKeys.userList(page);
      final cached = await _cache.get<Pagination<UserModel>>(cacheKey);
      if (cached != null) {
        return Result.success(cached);
      }

      final remote = await _remoteDataSource.fetchUserList(
        page: page,
        pageSize: pageSize,
      );

      await _cache.set(cacheKey, remote, duration: CacheConfig.getCacheDuration('user'));

      // 保存到本地
      for (final user in remote.data) {
        await _localDataSource.saveUser(user);
      }

      return Result.success(remote);
    } catch (e) {
      return Result.failure(e as Exception);
    }
  }

  @override
  Future<Result<UserModel>> updateUser(UserModel user) async {
    try {
      // 1. 更新远程
      final updated = await _remoteDataSource.updateUser(user);

      // 2. 更新本地
      await _localDataSource.saveUser(updated);

      // 3. 更新缓存
      final cacheKey = CacheKeys.user(user.id);
      await _cache.set(cacheKey, updated, duration: CacheConfig.getCacheDuration('user'));

      return Result.success(updated);
    } catch (e) {
      return Result.failure(e as Exception);
    }
  }
}
```

## 6. 数据同步

### 6.1 同步策略

```
┌────────────────────────────────────────┐
│         同步策略                       │
├────────────────────────────────────────┤
│  实时同步: WebSocket 推送              │

```

### 6.2 同步管理器

```dart
// core_database/lib/src/sync/data_sync_manager.dart
/// 数据同步管理器
class DataSyncManager {
  final AppDatabase _database;
  final ApiClient _apiClient;
  final Map<String, DateTime> _lastSyncTime = {};

  /// 增量同步
  Future<void> incrementalSync(String tableName) async {
    final lastSync = _lastSyncTime[tableName] ?? DateTime.fromMillisecondsSinceEpoch(0);
    final timestamp = lastSync.millisecondsSinceEpoch;

    try {
      // 从API获取增量数据
      final response = await _apiClient.get('/sync/$tableName', queryParameters: {
        'since': timestamp,
      });

      final dataList = response.data as List;

      if (dataList.isEmpty) return;

      // 批量更新到本地数据库
      final batch = _database.createBatch();
      for (var item in dataList) {
        // 使用 upsert (insert or update)
        batch.insert(tableName, item, mode: InsertMode.insertOrReplace);
      }
      await batch.commit();

      // 更新同步时间戳
      _lastSyncTime[tableName] = DateTime.now();

      AppLogger.i('增量同步成功: $tableName, 数量: ${dataList.length}');
    } catch (e) {
      AppLogger.e('增量同步失败: $tableName', error: e);
      rethrow;
    }
  }

  /// 全量同步
  Future<void> fullSync(String tableName) async {
    try {
      // 清空本地数据
      await _database.customStatement('DELETE FROM $tableName');

      // 分页拉取远程数据
      int page = 1;
      const pageSize = 100;
      int total = 0;

      while (true) {
        final response = await _apiClient.get('/$tableName', queryParameters: {
          'page': page,
          'page_size': pageSize,
        });

        final dataList = response.data['list'] as List;
        if (dataList.isEmpty) break;

        // 批量插入
        final batch = _database.createBatch();
        for (var item in dataList) {
          batch.insert(tableName, item);
        }
        await batch.commit();

        total += dataList.length;
        page++;

        // 如果数据少于页大小，说明已经是最后一页
        if (dataList.length < pageSize) break;
      }

      // 更新同步时间戳
      _lastSyncTime[tableName] = DateTime.now();

      AppLogger.i('全量同步成功: $tableName, 总数: $total');
    } catch (e) {
      AppLogger.e('全量同步失败: $tableName', error: e);
      rethrow;
    }
  }

  /// 清除同步记录
  void clearSyncRecord() {
    _lastSyncTime.clear();
  }
}
```

### 7.1 迁移策略

```dart
// 数据库迁移
onUpgrade: (Migrator m, int from, int to) async {
  // v1 → v2
  if (from == 1 && to == 2) {
    // 添加新字段
    await m.addColumn(users, users.level);
    await m.addColumn(users,_users.experience);

    // 创建新表
    await m.createTable(userLevels);

    // 数据迁移
    final allUsers = await select(users).get();
    for (final user in allUsers) {
      await into(userLevels).insert(UserLevelCompanion(
        userId: Value(user.id),
        level: const Value(1),
        experience: const Value(0),
      ));
    }
  }

  // v2 → v3
  if (from == 2 && to == 3) {
    // 添加索引
    await m.createIndex(users, 'index_users_created_at', [users.createdAt]);
    await m.createIndex(products, 'index_products_category', [products.categoryId]);
  }
}
```

## 8. 附录

### 8.1 数据库最佳实践

```
✅ 推荐做法:
   - 合理使用索引提升查询性能
   - 使用事务保证数据一致性
   - 批量操作使用 batch
   - 定期清理过期数据
   - 使用软删除而非物理删除

❌ 避免做法:
   - 在主线程进行数据库操作
   - 过度使用索引影响写入性能
   - 存储大文件到数据库
   - 忘记关闭数据库连接
```
