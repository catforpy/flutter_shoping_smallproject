## 1. 分层架构概述

### 1.1 为什么需要分层？

```
❌ 不分层的问题：
   ├── 代码耦合严重
   ├── 难以测试
   ├── 无法复用
   ├── 扩展困难
   └── 维护成本高

✅ 分层的好处：
   ├── 职责清晰
   ├── 易于测试
   ├── 高度复用
   ├── 扩展性强
   └── 维护成本低
```

### 1.2 分层原则

```
┌─────────────────────────────────────┐
│      上层依赖下层，单向依赖          │
│      下层不感知上层                 │
│      杜绝循环依赖                   │
└─────────────────────────────────────┘
```

### 1.3 六层架构图

```
┌─────────────────────────────────────────────┐
│  应用层 - 最终产品，可独立发布              │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  业务模块层 - 具体业务功能实现              │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  业务服务层 - 跨模块的共享业务逻辑          │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  核心能力层 - 通用技术能力                  │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  基础设施层 - 基础工具和类型定义            │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  平台适配层 - 平台相关功能封装              │
└─────────────────────────────────────────────┘
```

## 2. 应用层 (Apps)

### 2.1 层级定位

```
层级位置: 最顶层
依赖关系: 依赖业务模块层
包含内容: 4+ 个独立应用
```

### 2.2 职责定义

| 职责 | 说明 |
|------|------|
| **应用组装** | 组合各个业务模块，形成完整应用 |
| **主题配置** | 定义应用主题、样式、品牌色 |
| **路由配置** | 配置应用全局路由 |
| **入口管理** | 管理应用启动、生命周期 |
| **平台配置** | 配置 Android/iOS 原生设置 |

### 2.3 应用列表

| 应用 | 包名 | 说明 | 独立发布 |
|------|------|------|----------|
| **app_main** | com.yourcompany.app | 综合主应用 | ✅ |
| **app_shop** | com.yourcompany.shop | 电商独立应用 | ✅ |

// apps/app_main/lib/main.dart

  // 1. 初始化基础服务

  // 3. 启动应用

      child: MyApp(router: appRouter),

class MyApp extends StatelessWidget {

  @override
  Widget build(BuildContext context) {
    return MaterialApp.router(
      title: 'Your Super App',
      theme: AppTheme.lightTheme,
      darkTheme: AppTheme.darkTheme,
      routerConfig: router.router,

/// 应用组装 - 选择需要的模块
class AppRouter {
  static GoRouter createRouter() {
    return GoRouter(
      routes: [
        // 底部导航
        StatefulShellRoute.indexedStack(
          builder: (context, state, navigationShell) {
            return MainScreen(
              navigationShell: navigationShell,
            );
          },
          branches: [
            // 首页 - 使用电商模块
            ShellRoute(
              routes: [...ShopRouter.routes],
            ),
            // 直播 - 使用直播模块
            ShellRoute(
              routes: [...LiveRouter.routes],
            ),
            // IM - 使用 IM 模块
            ShellRoute(
              routes: [...IMRouter.routes],
            ),
            // 视频 - 使用视频模块
            ShellRoute(
              routes: [...VideoRouter.routes],
            ),
            // 我的 - 使用博客模块
            ShellRoute(
              routes: [...BlogRouter.routes],
            ),
          ],
        ),

  }
}

### 4.4 服务结构示例

```dart
// service_user/lib/src/user_service.dart
class UserService {
  final UserRepository _repository;
  final ApiClient _apiClient;
  final CacheManager _cache;

  UserService({
    required UserRepository repository,
    required ApiClient apiClient,
    required CacheManager cache,
  })  : _repository = repository,
        _apiClient = apiClient,
        _cache = cache;

  /// 获取用户信息
  Future<UserModel> getUser(String userId) async {
    // 1. 尝试从缓存获取
    final cached = await _cache.get<UserModel>('user_$userId');
    if (cached != null) {
      return cached;
    }

    // 2. 从数据库获取
    final local = await _repository.getUser(userId);
    if (local != null) {
      await _cache.set('user_$userId', local);
      return local;
    }

    // 3. 从远程获取
    final remote = await _apiClient.get('/users/$userId');
    final user = UserModel.fromJson(remote.data);

    // 4. 保存到本地和缓存
    await Future.wait([
      _repository.saveUser(user),
      _cache.set('user_$userId', user),
    ]);

    return user;
  }

  /// 更新用户信息
  Future<void> updateUser(UserModel user) async {
    // 1. 更新到远程
    await _apiClient.put('/users/${user.id}', data: user.toJson());

    // 2. 更新本地
    await _repository.updateUser(user);

    // 3. 更新缓存
    await _cache.set('user_${user.id}', user);
  }
}
```

// core_network/lib/src/dio_client.dart
class DioClient {
  late final Dio _dio;

  DioClient({
    required BaseOptions options,
    List<Interceptor> interceptors = const [],
  }) {
    _dio = Dio(options);

    // 添加拦截器
    _dio.interceptors.addAll([
      LogInterceptor(),
      TokenInterceptor(),
      CacheInterceptor(),
      ErrorInterceptor(),
      ...interceptors,
    ]);
  }

  Future<Response<T>> get<T>(
    String path, {
    Map<String, dynamic>? queryParameters,
    Options? options,
  }) async {
    try {
      return await _dio.get<T>(
        path,
        queryParameters: queryParameters,
        options: options,
      );
    } on DioException catch (e) {
      throw _handleError(e);
    }
  }

  // POST, PUT, DELETE, ...
}

/// String 扩展
extension StringExtension on String {
  /// 判断是否为空
  bool get isNotEmpty => this.isNotEmpty;

  /// 首字母大写
  String get capitalize {
    if (isEmpty) return this;
    return this[0].toUpperCase() + substring(1);
  }

  /// 移除所有空格
  String get removeAllSpaces => replaceAll(' ', '');

  /// 验证手机号
  bool get isPhoneNumber {
    return RegExp(r'^1[3-9]\d{9}$').hasMatch(this);
  }

  /// 验证邮箱
  bool get isEmail {
    return RegExp(r'^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$').hasMatch(this);
  }
}

## 8. 层间交互规则

### 8.1 依赖规则

```
✅ 允许的依赖：
   - 应用层 → 业务模块层
   - 业务模块层 → 业务服务层
   - 业务服务层 → 核心能力层
   - 核心能力层 → 基础设施层
   - 基础设施层 → 平台适配层

❌ 禁止的依赖：
   - 下层依赖上层
   - 同层相互依赖
   - 跨层依赖（除特殊情况）
   - 循环依赖
```

### 8.2 调用规则

```
┌──────────────────────────────────┐
│  UI 调用规则                      │
│  - UI 只能调用 Provider           │
│  - Provider 调用 UseCase          │
│  - UseCase 调用 Repository        │
│  - Repository 调用 DataSource     │
└──────────────────────────────────┘
```

### 8.3 数据流向

```
用户操作
   ↓
UI (Widget)
   ↓
Provider (Riverpod)
   ↓
UseCase (业务逻辑)
   ↓
Repository (数据仓库)
   ↓
DataSource (数据源)
   ↓
数据返回 (逐层返回)
```
