## 1. 项目根目录

### 1.1 完整目录树

```

│
├── packages/                    # 所有 Package
│   ├── core/                   # 核心能力层

│
├── tools/                       # 开发工具
│   ├── scripts/                # 构建脚本
│   ├── codegen/                # 代码生成
│   └── linter/                 # 代码检查
│
├── docs/                        # 项目文档
│   ├── architecture/           # 架构文档

│   ├── guides/                 # 开发指南
```

## 2. Package 目录结构

### 2.1 标准结构

```
package_name/
├── lib/
│   ├── package_name.dart       # 主导出文件
│   │
│   └── src/
│       ├── public/             # 公共 API
│       │   ├── *.dart         # 对外导出的类/函数
│       │   └── exports.dart    # 重新导出
│       │
│       ├── private/            # 私有实现
│       │   └── *_impl.dart    # 实现细节
│       │
│       └── generated/          # 自动生成代码
│
├── test/                       # 测试
│   ├── unit/                   # 单元测试
│   ├── widget/                 # Widget 测试
│   └── integration/            # 集成测试
│
├── android/                    # Android 平台代码（如果需要）
├── ios/                        # iOS 平台代码（如果需要）
├── web/                        # Web 平台代码（如果需要）
│
├── example/                    # 示例（可选）
│   ├── lib/
│   │   └── main.dart
│   └── pubspec.yaml
│
├── pubspec.yaml
├── README.md
└── CHANGELOG.md
```

## 4. 文件命名规范

### 4.1 命名规则

| 类型 | 规则 | 示例 |
|------|------|------|
| **Package** | 小写 + 下划线 | `core_network` |
| **文件** | 小写 + 下划线 | `user_service.dart` |
| **类** | 大驼峰 | `UserService` |
| **变量** | 小驼峰 | `userService` |
| **常量** | 小驼峰 | `defaultPageSize` |
| **私有成员** | 下划线前缀 | `_privateField` |
| **枚举** | 大驼峰 | `UserStatus` |
| **扩展方法** | 下划线后缀 | `string_extension.dart` |

### 4.2 特殊文件命名

```
主导出文件: package_name.dart
扩展方法: type_extension.dart (string_extension.dart)
用例: action_usecase.dart (get_user_usecase.dart)
页面: feature_page.dart (user_list_page.dart)
组件: feature_widget.dart (user_card_widget.dart)
状态: feature_provider.dart (user_provider.dart)
```

## 5. 目录组织原则

### 5.1 按功能分层

```
✅ 推荐: 按功能组织
module_shop/
├── presentation/
├── domain/
└── data/

❌ 不推荐: 按类型组织
module_shop/
├── pages/
├── widgets/
├── models/
└── services/
```

### 5.2 按职责分包

```
✅ 推荐: 单一职责
service_user/
service_order/
service_payment/

❌ 不推荐: 混合职责
service_business/  # 包含所有服务
```

### 5.3 公共代码提取

```
✅ 推荐: 提取公共代码
core_ui/          # 公共 UI 组件
core_utils/       # 公共工具
module_common/    # 公共业务模块

❌ 不推荐: 重复代码
module_shop/widgets/common_button.dart
module_live/widgets/common_button.dart
```

## 6. 配置文件

### 6.1 analysis_options.yaml

```yaml
# 项目根目录
include: package:very_good_analysis/analysis_options.yaml

analyzer:
  plugins:
    - dart_code_metrics

  exclude:
    - '**/*.g.dart'
    - '**/*.freezed.dart'
    - '**/generated/**'

  errors:
    invalid_annotation_target: ignore
    missing_required_param: error
    missing_return: error

linter:
  rules:
    # 强制规则
    - prefer_const_constructors
    - prefer_const_declarations
    - avoid_print
    - avoid_unnecessary_containers
```

### 6.2 melos.yaml

```yaml
name: your_super_app

packages:
  - packages/**
  - apps/**

command:
  bootstrap:
    runPubGetInParallel: false

scripts:
  # 开发
  start:
    run: cd apps/app_main && flutter run

  # 构建
  build:
    run: melos run build:apk

  build:apk:
    run: cd apps/app_main && flutter build apk --release

  # 测试
  test:
    run: melos exec -- dart test

  # 代码生成
  gen:
    run: melos exec -c 1 -- dart run build_runner build

  # 格式化
  format:
    run: dart format .

  # 分析
  analyze:
    run: melos exec -c 1 -- dart analyze
```
