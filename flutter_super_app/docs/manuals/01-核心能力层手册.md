## 1. core_network - 网络请求

### 📍 路径
```
packages/core/core_network/
```

### 🎯 功能概述
封装所有 HTTP 请求功能，提供统一的网络请求接口。

### 📄 文件结构
```
lib/
├── core_network.dart              # 主导出文件
└── src/
    ├── clients/
    │   ├── http_client.dart        # HTTP 客户端（底层）
    │   └── api_client.dart         # API 客户端（高层）
    ├── interceptors/
    │   ├── interceptor.dart        # 拦截器接口
    │   ├── log_interceptor.dart    # 日志拦截器
    │   ├── auth_interceptor.dart   # 认证拦截器
    │   └── error_interceptor.dart  # 错误拦截器
    └── models/
        ├── request_options.dart    # 请求选项
        └── response.dart           # 响应模型
```

### 🔧 核心组件详解

#### 1.1 HttpClient - HTTP 客户端（底层）

**位置**：`lib/src/clients/http_client.dart`

**功能**：封装 `http` 包，提供基础的 HTTP 请求功能

**核心属性**：
```dart
baseUrl: String                    // 基础 URL
interceptors: List<Interceptor>    // 拦截器列表
```

**核心方法**：
```dart
// 通用请求方法
request(RequestOptions options) → Future<ApiResponse<dynamic>>

// 快捷方法
get(String path, {...}) → Future<ApiResponse<dynamic>>
post(String path, {body, ...}) → Future<ApiResponse<dynamic>>
put(String path, {body, ...}) → Future<ApiResponse<dynamic>>
delete(String path, {body, ...}) → Future<ApiResponse<dynamic>>
```

**⬇️ 内部调用**：
- 使用 `http` 包
- 调用拦截器链

**⬆️ 被调用**：
- ApiClient

---

#### 1.2 ApiClient - API 客户端（高层）

**位置**：`lib/src/clients/api_client.dart`

**功能**：基于 HttpClient，提供更高级的 API 请求功能

**核心属性**：
```dart
baseUrl: String         // 基础 URL
```

**核心方法**：
```dart
// 泛型方法，自动解析响应数据
get<T>(String path, {
  Map<String, String>? headers,
  Map<String, dynamic>? queryParameters,
  bool requireAuth = true,
  T Function(dynamic)? dataParser,  // 自定义数据解析器
}) → Future<ApiDataResponse<T>>

post<T>(String path, {
  Map<String, String>? headers,
  Map<String, dynamic>? queryParameters,
  dynamic body,
  bool requireAuth = true,
  T Function(dynamic)? dataParser,
}) → Future<ApiDataResponse<T>>

// ... put, delete 同理
```

**⬇️ 内部调用**：
- 使用 `HttpClient`
- 调用拦截器

**⬆️ 被调用**：
- 所有需要网络请求的业务服务（service_user, service_auth 等）

**使用示例**：
```dart
// 创建 API 客户端
final httpClient = HttpClient(
  baseUrl: 'https://api.example.com',
  interceptors: [
    LogInterceptor(),
    AuthInterceptor(tokenGetter: () => getToken()),
  ],
);

final apiClient = ApiClient(
  httpClient: httpClient,
);

// 使用
final response = await apiClient.get<User>(
  '/user/123',
  dataParser: (json) => User.fromJson(json),
);

if (response.success) {
  final user = response.data;
  print(user.name);
}
```

#### 1.3 Interceptor - 拦截器系统

**位置**：`lib/src/interceptors/interceptor.dart`

**功能**：在请求发送前后和错误发生时执行自定义逻辑

**拦截器接口**：
```dart
abstract base class Interceptor {
  // 请求拦截（可修改请求参数）
  Future<RequestOptions> onRequest(RequestOptions options)

  // 响应拦截（可修改响应数据）
  Future<ApiResponse<dynamic>> onResponse(ApiResponse<dynamic> response)

  // 错误拦截（可处理错误）
  Future<dynamic> onError(dynamic error)
}
```

**拦截器执行顺序**：
```
请求：
Request → onRequest → onRequest → ... → 实际发送

响应：
实际接收 → onResponse → onResponse → ... → 返回

错误：
发生错误 → onError → onError → ... → 抛出/处理
```

**⬇️ 内部调用**：
- 可使用任何核心层能力

**⬆️ 被调用**：
- HttpClient（在请求流程中调用）

##### 1.4.2 AuthInterceptor - 认证拦截器

**位置**：`lib/src/interceptors/auth_interceptor.dart`

**功能**：自动在请求头中添加 Token

**核心属性**：
```dart
tokenGetter: String? Function()           // 获取 Token
tokenRefresher: Future<String?> Function() // 刷新 Token
onTokenExpired: void Function()            // Token 失效回调
```

**使用场景**：需要身份认证的 API

**使用示例**：
```dart
AuthInterceptor(
  tokenGetter: () {
    // 从本地存储获取 Token
    return SharedPreferences.getString('token');
  },
  tokenRefresher: () async {
    // Token 失效时，自动刷新
    final newToken = await api.refreshToken();
    return newToken;
  },
  onTokenExpired: () {
    // Token 刷新失败，跳转到登录页
    router.push('/login');
  },
);
```

**流程**：
```
1. 请求前：检查 options.requireAuth
2. 如果需要认证：调用 tokenGetter() 获取 Token
3. 添加到请求头：Authorization: Bearer <token>
4. 响应后：检查状态码是否为 401
5. 如果是 401：尝试刷新 Token，或触发 onTokenExpired
```

**⬇️ 内部调用**：
- 可能调用 tokenRefresher（网络请求）

**⬆️ 被调用**：
- 添加到 HttpClient 的拦截器链中

---

##### 1.4.3 ErrorInterceptor - 错误拦截器

**位置**：`lib/src/interceptors/error_interceptor.dart`

**功能**：统一处理 HTTP 错误状态码

**核心逻辑**：
```dart
if (!response.isSuccess) {
  throw NetworkException.fromStatusCode(
    response.statusCode,
    response.statusMessage,
  );
}
```

**错误映射**：
```
400 → BadRequestException
401 → UnauthorizedException
403 → ForbiddenException
404 → NotFoundException
408 → RequestTimeoutException
429 → TooManyRequestsException
500 → ServerException
502 → BadGatewayException
503 → ServiceUnavailableException
504 → GatewayTimeoutException
其他 → HttpException
```

**使用场景**：统一错误处理

**⬇️ 内部调用**：
- 使用 `core_exceptions` 的异常类

**⬆️ 被调用**：
- 添加到 HttpClient 的拦截器链中（通常是最后一个）

    ↓

    ↓

### 🎯 使用流程

#### 场景：发起一个 API 请求

```dart
// 1. 创建 HTTP 客户端，配置拦截器
final httpClient = HttpClient(
  baseUrl: ApiConstants.baseUrl,
  interceptors: [
    LogInterceptor(logBody: true),
    AuthInterceptor(
      tokenGetter: () => localStorage.get('token'),
    ),
    ErrorInterceptor(),
  ],
);

// 2. 创建 API 客户端
final apiClient = ApiClient(httpClient: httpClient);

// 3. 发起请求
try {
  final response = await apiClient.get<User>(
    ApiConstants.endpoints.userInfo,
    dataParser: (json) => User.fromJson(json),
  );

  if (response.success) {
    final user = response.data;
    print('用户名：${user.name}');
  } else {
    print('错误：${response.message}');
  }
} catch (e) {
  // 错误已被 ErrorInterceptor 转换为 NetworkException
  print('请求失败：$e');
}
```

### 📝 常见问题

#### Q1: 如何添加自定义拦截器？

```dart
class CustomInterceptor extends BaseInterceptor {
  @override
  Future<RequestOptions> onRequest(RequestOptions options) async {
    // 在请求前执行
    print('发送请求：${options.url}');
    return options;
  }

  @override
  Future<ApiResponse<dynamic>> onResponse(
    ApiResponse<dynamic> response
  ) async {
    // 在响应后执行
    print('收到响应：${response.statusCode}');
    return response;
  }
}

// 使用
final client = HttpClient(
  interceptors: [CustomInterceptor()],
);
```

#### Q2: 如何处理分页数据？

```dart
final response = await apiClient.get<List<User>>(
  '/users',
  queryParameters: {
    'page': '1',
    'pageSize': '20',
  },
  dataParser: (json) {
    final list = json as List;
    return list.map((e) => User.fromJson(e)).toList();
  },
);

// 配合 Pagination 使用
final pagination = Pagination(
  page: 1,
  pageSize: 20,
  total: response.data.length,
);

final pagedResult = PagedResult(
  data: response.data,
  pagination: pagination,
);
```

#### Q3: 如何上传文件？

```dart
final response = await apiClient.post<Map>(
  ApiConstants.endpoints.upload,
  body: FormData.fromMap({
    'file': MultipartFile.fromFile(
      filePath,
      filename: 'image.jpg',
    ),
  }),
);
```
