# 调用关系图谱

> 本手册追踪所有重要方法之间的调用关系，帮助你理解代码的网状调用结构

---

## 📖 如何使用本图谱

### 图例说明

```
⬇️  调用下层（我调用了谁）
⬆️  被上层调用（谁调用了我）
🔄 相互调用
```

### 查阅方法

1. **自上而下**：从高层开始，逐层查看调用了哪些底层方法
2. **自下而上**：从底层开始，逐层查看被哪些上层方法调用
3. **搜索方法名**：直接搜索你想了解的方法名

## 1. 用户登录流程调用链

### 1.1 完整调用链

```
UI层 (点击登录按钮)
    ↓
┌──────────────────────────────────────┐
│ service_auth (认证服务)               │
│ └─ login(String phone, String pwd)   │
└──────────────────────────────────────┘
    ↓
┌──────────────────────────────────────┐
│ core_network (网络请求)              │
│ ├─ HttpClient.post()                 │
│ ├─ AuthInterceptor (添加 Token)     │
│ ├─ LogInterceptor (记录日志)        │
│ └─ ErrorInterceptor (处理错误)      │
└──────────────────────────────────────┘
    ↓
┌──────────────────────────────────────┐
│ core_utils (工具函数)                │
│ ├─ StringExtension.isPhoneNumber()  │
│ └─ Validator.isPassword()           │
└──────────────────────────────────────┘
    ↓
┌──────────────────────────────────────┐
│ core_exceptions (异常处理)           │
│ └─ NetworkException                 │
└──────────────────────────────────────┘
    ↓
┌──────────────────────────────────────┐
│ core_constants (常量)                │
│ └─ ApiConstants.endpoints.login      │
└──────────────────────────────────────┘
```

### 1.2 方法级调用链

#### service_user.login()

```dart
// 文件：packages/services/service_user/lib/user_service.dart

Future<Result<User>> login(String phone, String password) async {
  // ⬇️ 调用下层

  // 1. 验证参数
  if (!phone.isPhoneNumber) {  // ⬇️ core_utils/StringExtension
    return Result.failure(
      ValidationException(message: '手机号格式错误')
    );  // ⬇️ core_exceptions
  }

  // 2. 发起网络请求
  final response = await _apiClient.post(  // ⬇️ core_network/ApiClient
    ApiConstants.endpoints.login,  // ⬇️ core_constants
    body: {
      'phone': phone,
      'password': password,
    },
  );

  // 3. 记录日志
  Log.i('登录成功');  // ⬇️ core_logging

  // 4. 保存 Token
  await _storage.setString(  // ⬇️ core_cache
    StorageKeys.accessToken,  // ⬇️ core_constants
    response.data['token'],
  );

  return Result.success(User.fromJson(response.data));
}
```

**调用关系总结**：
```
service_user.login()
    ├─ StringExtension.isPhoneNumber()     [core_utils]
    ├─ ValidationException                 [core_exceptions]
    ├─ ApiClient.post()                     [core_network]
    │   ├─ HttpClient.request()             [core_network]
    │   ├─ AuthInterceptor.onRequest()     [core_network]
    │   ├─ LogInterceptor.onRequest()      [core_network]
    │   ├─ ErrorInterceptor.onResponse()   [core_network]
    │   └─ NetworkException                [core_exceptions]
    ├─ ApiConstants.endpoints.login        [core_constants]
    ├─ Log.i()                             [core_logging]
    └─ StorageKeys.accessToken            [core_constants]
```

## 2. 网络请求流程调用链

### 2.1 ApiClient.get() 调用链

```dart
// 文件：packages/core/core_network/lib/src/clients/api_client.dart

Future<ApiDataResponse<T>> get<T>(String path, {
  Map<String, String>? headers,
  Map<String, dynamic>? queryParameters,
  bool requireAuth = true,
  T Function(dynamic)? dataParser,
}) async {
  // ⬇️ 调用下层

  // 1. 调用 HttpClient.get()
  final response = await _httpClient.get(  // ⬇️ core_network/HttpClient
    path,
    headers: headers,
    queryParameters: queryParameters,
    requireAuth: requireAuth,
  );

  // 2. 解析响应
  return _parseResponse<T>(response, dataParser);
}

ApiDataResponse<T> _parseResponse<T>(
  ApiResponse response,      // ⬇️ core_network/ApiResponse
  T Function(dynamic)? dataParser,
) {
  // 3. 解析 JSON
  if (response.data is Map<String, dynamic>) {
    return ApiDataResponse.fromJson(  // ⬇️ core_network/ApiDataResponse
      response.data as Map<String, dynamic>,
      dataParser: dataParser,
    );
  }

  return ApiDataResponse<T>(...);
}
```

**调用关系总结**：
```
ApiClient.get()
    ├─ HttpClient.get()                    [core_network]
    │   ├─ HttpClient.request()            [core_network]
    │   │   ├─ Interceptor.onRequest()     [core_network] (循环)
    │   │   ├─ _buildFullUrl()             [内部方法]
    │   │   ├─ _performRequest()           [内部方法]
    │   │   │   └─ http.Client.get/post/... [http 包]
    │   │   ├─ _parseResponseData()        [内部方法]
    │   │   └─ Interceptor.onResponse()    [core_network] (循环)
    │   └─ ApiResponse()                   [core_network]
    └─ ApiDataResponse.fromJson()         [core_network]
```

### 2.2 HttpClient.request() 拦截器调用链

```dart
// 文件：packages/core/core_network/lib/src/clients/http_client.dart

Future<ApiResponse<dynamic>> request(RequestOptions options) async {
  // 1. 请求拦截器链（循环）
  var processedOptions = options;
  for (final interceptor in interceptors) {
    processedOptions = await interceptor.onRequest(processedOptions);
    // ⬇️ 可能调用的拦截器：
    // - LogInterceptor.onRequest()     [core_network]
    // - AuthInterceptor.onRequest()    [core_network]
  }

  // 2. 发起 HTTP 请求
  final response = await _performRequest(...);

  // 3. 错误处理（如果出错）
  try { } catch (e) {
    for (final interceptor in interceptors) {
      await interceptor.onError(e);  // ⬇️ ErrorInterceptor
    }
    rethrow;
  }

  // 4. 响应拦截器链（循环）
  var processedResponse = apiResponse;
  for (final interceptor in interceptors) {
    processedResponse = await interceptor.onResponse(processedResponse);
    // ⬇️ 可能调用的拦截器：
    // - ErrorInterceptor.onResponse()  [core_network]（可能抛异常）
    // - LogInterceptor.onResponse()     [core_network]
  }

  return processedResponse;
}
```

**拦截器执行顺序**：
```
请求：
LogInterceptor.onRequest()     → 记录日志
    ↓
AuthInterceptor.onRequest()    → 添加 Token
    ↓
实际发送请求

响应：
实际接收响应
    ↓
ErrorInterceptor.onResponse()  → 检查错误码，可能抛异常
    ↓
LogInterceptor.onResponse()   → 记录响应
    ↓
返回数据
```

## 3. 数据验证流程调用链

### 3.1 Validator.isPassword() 调用链

```dart
// 文件：packages/core/core_utils/lib/src/utils/validator.dart

static bool isPassword(String? value) {
  // ⬇️ 调用下层扩展方法

  if (value.isNullOrEmpty) return false;  // ⬇️ StringExtension
  if (value!.length < 8 || value.length > 20) return false;

  // ⬇️ 使用正则表达式
  final hasLetter = RegExp(r'[a-zA-Z]').hasMatch(value);
  final hasNumber = RegExp(r'\d').hasMatch(value);

  return hasLetter && hasNumber;
}
```

**调用关系**：
```
Validator.isPassword()
    ├─ StringExtension.isNullOrEmpty()  [core_utils]
    └─ RegExp.hasMatch()                [Dart 核心]
```

## 4. 异常处理流程调用链

### 4.1 ErrorInterceptor.onResponse() 调用链

```dart
// 文件：packages/core/core_network/lib/src/interceptors/error_interceptor.dart

Future<ApiResponse<dynamic>> onResponse(
  ApiResponse<dynamic> response
) async {
  // ⬇️ 调用下层

  if (!response.isSuccess) {
    throw NetworkException.fromStatusCode(  // ⬇️ core_exceptions
      response.statusCode,
      response.statusMessage,
    );
    // ⬇️ 返回的异常可能是：
    // - BadRequestException
    // - UnauthorizedException
    // - NotFoundException
    // - ServerException
    // ...
  }

  return response;
}
```

**调用关系**：
```
ErrorInterceptor.onResponse()
    ├─ ApiResponse.isSuccess            [core_network]
    └─ NetworkException.fromStatusCode()  [core_exceptions]
        └─ 返回具体异常子类
            ├─ BadRequestException()
            ├─ UnauthorizedException()
            └─ ...
```

## 5. 日志记录流程调用链

### 5.1 Log.i() 调用链

```dart
// 文件：packages/core/core_logging/lib/src/loggers/app_logger.dart

static void i(String message, {
  Object? error,
  StackTrace? stackTrace,
}) {
  root.info(message, error: error, stackTrace: stackTrace);
}

// AppLogger.info()
void info(String message, {
  Object? error,
  StackTrace? stackTrace,
}) {
  log(LogLevel.info, message, error: error, stackTrace: stackTrace);
}

// AppLogger.log()
void log(
  LogLevel level,
  String message, {
  Object? error,
  StackTrace? stackTrace,
}) {
  if (!level.shouldLog(minLevel)) return;  // ⬇️ LogLevel

  final timestamp = DateTime.now();
  final logMessage = '[$timestamp] [$level] [$tag] $message';

  // ⬇️ 使用 developer.log 和 print
  developer.log(
    logMessage,
    name: tag,
    level: level.value,
    error: error,
    stackTrace: stackTrace,
    time: timestamp,
  );

  print(logMessage);  // ⬇️ Dart 核心
}
```

**调用关系**：
```
Log.i()
    ├─ Log.root                          [core_logging]
    ├─ AppLogger.info()                  [core_logging]
    ├─ AppLogger.log()                   [core_logging]
    ├─ LogLevel.shouldLog()              [core_logging]
    ├─ DateTime.now()                    [Dart 核心]
    ├─ developer.log()                   [Dart 核心]
    └─ print()                           [Dart 核心]
```

## 8. 错误处理流程调用链

### 8.1 完整流程

```
网络请求失败
    ↓
抛出原始异常（如 SocketException）
    ↓
拦截器 onError() 捕获
    ↓
重新抛出异常
    ↓
上层 try-catch 捕获
    ↓
转换为业务异常
    ↓
显示给用户
```

**调用关系**：
```
try {
  apiClient.get(...);
} catch (e) {
  // ⬇️ 判断异常类型
  if (e is NetworkException) {
    // ⬇️ 转换为 UI 友好消息
    showError(e.getUserMessage());
  } else if (e is AuthException) {
    // ⬇️ 跳转到登录页
    router.push('/login');
  }
}
```

## 🎯 开发建议

### 1. 理解调用链的技巧

**技巧 1**：从高层开始追踪
```
从 UI 层的业务方法开始（如 service_user.login）
    → 逐层向下追踪
    → 看每个方法调用了哪些下层方法
```

**技巧 2**：从底层开始追溯
```
从底层方法开始（如 StringExtension.isPhoneNumber）
    → 搜索哪些地方调用了它
    → 理解它在整个系统中的作用
```

**技巧 3**：关注拦截器
```
拦截器是调用链的"横切点"
它们可以在不修改原有代码的情况下
为所有请求添加额外功能（日志、认证、错误处理）
```

### 2. 调试技巧

#### 技巧 1：追踪网络请求

```dart
// 在 LogInterceptor 中打断点
@override
```

#### 技巧 2：追踪异常

## 📝 调用关系维护

### 添加新方法时

1. **更新本手册**：在对应章节添加调用链说明
2. **标注依赖**：明确标注 ⬇️ 内部调用 和 ⬆️ 被调用
3. **提供示例**：给出完整的使用示例和调用关系

### 修改方法时

1. **检查影响范围**：搜索哪些地方调用了这个方法
2. **更新调用链**：修改本手册中的调用关系
3. **测试验证**：确保所有调用方都能正常工作
