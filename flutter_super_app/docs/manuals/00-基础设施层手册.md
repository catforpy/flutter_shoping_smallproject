## 1. core_base - 基础类型

### 📍 路径
```
packages/core/core_base/
```

### 🎯 功能概述
提供项目中最基础的数据类型，所有其他包都可能使用这些类型。

### 📄 文件结构
```
lib/
├── core_base.dart              # 主导出文件
└── src/
    ├── types/
    │   ├── result.dart         # Result 类型（成功/失败）
    │   ├── either.dart         # Either 类型（两者之一）
    │   ├── pagination.dart     # 分页类型
    │   └── paged_result.dart   # 分页结果
    └── entities/
        └── base_entity.dart    # 基础实体类
```

### 🔧 核心类型详解

#### 1.1 Result<T> - 操作结果类型

**用途**：表示一个操作的结果，可能是成功或失败

**位置**：`lib/src/types/result.dart`

**核心方法**：
```dart
// 创建成功结果
Result.success<T>(T value)

// 创建失败结果
Result.failure<T>(Exception error)

// 获取值或抛出异常
valueOrThrow → T

// 获取值或返回默认值
getOrElse(T defaultValue) → T

// 映射值
map<R>(R Function(T) mapper) → Result<R>
```

**⬇️ 内部调用**：无（最底层类型）

**⬆️ 被调用**：
- 所有需要返回结果的场景（网络请求、数据库操作等）

**使用示例**：
```dart
// 网络请求场景
Future<Result<User>> getUser(String id) async {
  try {
    final user = await api.getUser(id);
    return Result.success(user);
  } catch (e) {
    return Result.failure(Exception('获取用户失败'));
  }
}

// 使用
final result = await getUser('123');
if (result.isSuccess) {
  print(result.valueOrThrow.name);
}
```

---

#### 1.2 Either<L, R> - 两种可能值

**用途**：表示两种可能值之一，常用于错误处理（Left=错误，Right=成功）

**位置**：`lib/src/types/either.dart`

**核心方法**：
```dart
// 创建 Left（通常表示错误）
Either.left<L, R>(L value)

// 创建 Right（通常表示成功）
Either.right<L, R>(R value)

// 折叠
fold<T>(T Function(L) onLeft, T Function(R) onRight) → T

// 链式调用
flatMap<R>(Either<L, R> Function(R) mapper) → Either<L, R>
```

**⬇️ 内部调用**：无

**⬆️ 被调用**：
- 需要细粒度错误处理的场景

**使用示例**：
```dart
Either<String, User> getUser(String id) {
  final user = database.findUser(id);
  if (user == null) {
    return Either.left('用户不存在');
  }
  return Either.right(user);
}
```

#### 1.3 Pagination - 分页信息

**核心方法**：

**⬇️ 内部调用**：无

**⬆️ 被调用**：

#### 1.4 PagedResult<T> - 分页结果

**用途**：包装分页数据和分页信息

**位置**：`lib/src/types/paged_result.dart`

**核心属性**：
```dart
data: List<T>              // 数据列表
pagination: Pagination     // 分页信息
```

**核心方法**：
```dart
map<R>(R Function(T) mapper) → PagedResult<R>  // 转换数据类型
```

**⬇️ 内部调用**：
- 使用 `Pagination`

**⬆️ 被调用**：
- 网络层返回分页数据

**使用示例**：
```dart
// 网络请求返回
Future<PagedResult<User>> getUsers(int page) async {
  final response = await api.get('/users', page: page);
  return PagedResult(
    data: response.data,
    pagination: Pagination(
      page: page,
      pageSize: 20,
      total: response.total,
    ),
  );
}
```

enum LogLevel {
  debug(0),   // 调试信息

### 🔧 AppLogger - 日志记录器

**位置**：`lib/src/loggers/app_logger.dart`

**核心方法**：
```dart
// 创建日志记录器
AppLogger(String tag, {LogLevel minLevel})

// 记录不同级别的日志
debug(String message, {Object? error, StackTrace? stackTrace})
info(String message, {Object? error, StackTrace? stackTrace})
warning(String message, {Object? error, StackTrace? stackTrace})
error(String message, {Object? error, StackTrace? stackTrace})
fatal(String message, {Object? error, StackTrace? stackTrace})
```

**使用示例**：
```dart
// 创建日志记录器
final logger = AppLogger('UserService', minLevel: LogLevel.debug);

// 记录日志
logger.debug('开始获取用户信息');
logger.info('用户信息获取成功');
logger.warning('用户信息不完整');
logger.error('获取用户信息失败', error: e);

// 全局快捷方法
Log.i('应用启动');
Log.e('发生错误', error: exception);
```

## 📝 开发建议

1. **优先使用扩展方法**：如 `string.isPhoneNumber` 而不是 `Validator.isPhoneNumber(string)`
2. **统一使用异常类**：不要抛出原始 Exception，而是使用定义好的异常类
3. **使用常量**：不要使用魔法值，从 `core_constants` 中获取
4. **记录日志**：在关键操作前后记录日志
