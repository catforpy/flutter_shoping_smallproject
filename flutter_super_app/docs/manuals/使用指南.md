> 告诉你如何快速上手这个手册系统，快速融入项目架构

---

## 🎯 这个手册系统是做什么的？

### 解决的问题

你在做一个复杂的 Flutter 项目，遇到了这些问题：

1. **不知道某个方法在哪里**：想用格式化日期的方法，但不知道在哪
2. **不懂方法的调用关系**：看到 `ApiClient.get()`，不知道它内部调用了什么
3. **不知道某个包是干什么的**：看到 `core_utils`，不知道它提供什么功能
4. **代码调用太复杂，看不懂**：一个方法调用了5层，不知道怎么追踪
5. **想快速查阅，不想翻代码**：需要一个"字典"，能快速查找

**这个手册系统就是为了解决这些问题！**

   ```

   跳转到 "00-基础设施层手册.md" 的 core_utils 章节

4. **查看完整文档**
   ```
   ✅ 方法签名
   ✅ 功能描述
   ✅ 参数说明
   ✅ 返回值说明
   ✅ 使用示例
   ✅ 调用关系（谁调用了我，我调用了谁）

### 场景 2：我想理解一个方法的调用链

**例子**：`ApiClient.get()` 内部调用了什么？

**步骤**：

1. **打开调用关系图谱** → `03-调用关系图谱.md`

2. **搜索 "ApiClient.get"**

3. **查看完整的调用链**
   ```
   ApiClient.get()
       ├─ HttpClient.get()              [core_network]
       │   ├─ HttpClient.request()      [core_network]
       │   │   ├─ 拦截器链              [core_network]
       │   │   └─ http.Client.get()     [http 包]
       │   └─ ApiResponse()             [core_network]
   ```

4. **逐层深入**
   ```
   点击 "HttpClient.request()"
   → 看到拦截器如何执行
   → 看到 HTTP 请求如何发送
   ```

**耗时**：1分钟

## 💡 使用技巧

### 技巧 1：逐层理解

**错误做法**：
```
❌ 直接看代码，看不懂就放弃
```

**正确做法**：
```
✅ 先看手册，理解整体结构
✅ 再看调用关系，理解调用链
✅ 最后看代码，理解实现细节
```

**例子**：理解 `ApiClient.get()`

1. **第一层**：看快速查询表，知道这个方法是做什么的
   ```
   ApiClient.get() → 发起 GET 请求
   ```

2. **第二层**：看调用关系图谱，知道它内部调用了什么
   ```
   ApiClient.get()
       ↓
   HttpClient.get()
       ↓
   拦截器链
   ```

3. **第三层**：看核心能力层手册，了解详细用法
   ```
   参数说明、返回值、使用示例
   ```

4. **第四层**：看代码，理解具体实现

### 技巧 3：建立心智模型

**什么是心智模型**：
```
你对代码结构的整体认知
就像地图一样，知道东西大概在哪里
```

**如何建立**：
1. **先看整体**：阅读 README.md，了解整体结构
2. **再看部分**：阅读各层手册，了解每层职责
3. **后看细节**：阅读调用关系图谱，了解调用链
4. **实践验证**：写代码时，对照手册，验证理解

**心智模型示例**：
```
基础设施层（地基）
    ├─ core_base: 基础类型（Result, Either, Pagination）
    ├─ core_utils: 工具函数（验证、格式化）
    ├─ core_constants: 常量（API地址、配置）
    ├─ core_exceptions: 异常（网络异常、认证异常）
    └─ core_logging: 日志（记录调试信息）

核心能力层（工具）
    ├─ core_network: 网络请求（API 调用）
    ├─ core_database: 数据库（本地存储）
    ├─ core_cache: 缓存（临时存储）
    └─ ...

业务服务层（业务）
    ├─ service_user: 用户服务
    ├─ service_auth: 认证服务
    └─ ...

应用层（产品）
    └─ app_main: 主应用
```

---

### 技巧 4：追踪调用链

**场景**：调试网络请求时，想知道日志在哪里打印

**步骤**：

1. **打开调用关系图谱** → `03-调用关系图谱.md`

2. **搜索 "ApiClient.get"**
   ```
   找到 ApiClient.get() 的调用链
   ```

3. **查看拦截器部分**
   ```
   看到 LogInterceptor.onRequest()
   → 这就是打印请求日志的地方
   ```

4. **跳转到代码**
   ```
   直接打开 LogInterceptor 的文件
   找到 onRequest 方法
   添加断点或打印语句
   ```

**调用关系图谱的价值**：
```
❌ 没有图谱：
   不知道从哪开始找
   不知道哪些拦截器会执行
   不知道日志在哪里打印

✅ 有图谱：
   一眼看出拦截器执行顺序
   快速定位到 LogInterceptor
   知道日志在拦截器中打印
```
