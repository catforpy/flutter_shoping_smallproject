## 🔍 常见调用模式

### 模式 1：网络请求标准流程

```dart
// 1. 创建 API 客户端
final apiClient = ApiClient(
  httpClient: HttpClient(
    baseUrl: ApiConstants.baseUrl,
    interceptors: [
      LogInterceptor(),
      AuthInterceptor(tokenGetter: () => token),
      ErrorInterceptor(),
    ],
  ),
);

// 2. 发起请求
final response = await apiClient.get<User>(
  ApiConstants.endpoints.userInfo,
  dataParser: (json) => User.fromJson(json),
);

// 3. 处理结果
if (response.success) {
  final user = response.data;
  Log.i('获取用户成功：${user.name}');
} else {
  Log.e('获取用户失败：${response.message}');
}
```

**调用链**：
```
ApiClient.get()
    → HttpClient.get()
    → Interceptor.onRequest() × 3
    → http.Client.get()
    → Interceptor.onResponse() × 3
    → ApiDataResponse.fromJson()
    → User.fromJson()
```

### 模式 2：数据验证标准流程

```dart
// 1. 验证手机号
if (!phone.isPhoneNumber) {
  throw const ValidationException(
    message: '手机号格式不正确',
  );
}

// 2. 验证密码
if (!password.isPassword) {
  throw const ValidationException(
    message: '密码必须包含字母和数字',
  );
}

// 3. 综合验证
final result = Validator.validate(
  phone: phone,
  password: password,
);

if (!result.isValid) {
  showErrors(result.errors);
}
```

**调用链**：
```
StringExtension.isPhoneNumber
    → RegExp.hasMatch()

Validator.isPassword
    → StringExtension.isNullOrEmpty
    → RegExp.hasMatch()

Validator.validate
    → Validator.isPhoneNumber
    → Validator.isEmail
    → ...
```

### 模式 3：异常处理标准流程

```dart
try {
  final response = await apiClient.get('/user/123');

  if (response.success) {
    return Result.success(response.data);
  } else {
    throw NetworkException.fromStatusCode(
      response.statusCode,
      response.message,
    );
  }
} on NetworkException catch (e) {
  // 已被 ErrorInterceptor 转换的具体异常
  Log.e('网络请求失败', error: e);
  return Result.failure(e);
} on Exception catch (e) {
  // 其他未预期的异常
  Log.e('未知错误', error: e);
  return Result.failure(e);
}
```

**调用链**：
```
try-catch
    → ApiClient.get()
    → HttpClient.get()
    → ErrorInterceptor.onResponse()
    → NetworkException.fromStatusCode()
    → 返回具体异常子类
    → Log.e()
    → Result.failure()
```
